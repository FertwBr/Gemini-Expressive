# Contributing to Gemini Expressive

Thank you for your interest in contributing to Gemini Expressive! Community contributions help make this extension better for everyone.

Please take a moment to review these guidelines before getting started.

## 🗺️ How to Contribute

### 1. Finding Issues
Look for issues labeled with:
* `good first issue`: Ideal for newcomers or minor quick fixes.
* `help wanted`: Tasks that need extra attention or community support.

### 2. Branching Strategy
We follow a feature-branching workflow. When working on a fix or a new feature, please create a specific branch from `main` using the following prefix convention:
* `feat/your-feature-name`: For new features or visual enhancements.
* `fix/bug-name`: For fixing broken logic or layout glitches.
* `docs/documentation-topic`: For editing README, inline comments, or guides.
* `test/suite-name`: For adding or modifying automated unit tests.

### 3. Development Setup
To set up the project locally on your machine:
